# BITS-WILP-Dissertation
BITS WILP Dissertation - M.Tech Software Engineering
ABSTRACT : 
In Agile software development environments, release decisions are often taken under strict timelines and changing business requirements. Teams usually rely on multiple disconnected metrics such as sprint completion status, defect count, testing progress, and build stability to determine whether a software release is ready for deployment. Since these indicators are spread across different tools and reports, release evaluation becomes difficult and sometimes subjective.
This dissertation presents the design and implementation of a Release Readiness Analytics Dashboard that provides a consolidated and quantitative assessment of release readiness for Agile software projects. The proposed framework integrates with Jira Cloud using REST APIs to collect sprint and issue-related information. Important software engineering metrics such as story completion rate, test coverage, open critical defects, build success rate, must-have feature completion, and stakeholder demo feedback are evaluated using a weighted scoring model.
The system is implemented using Flask for backend services and Streamlit for interactive dashboard visualisation. The framework classifies sprint readiness into READY, CONDITIONAL, and NOT READY states based on configurable thresholds. The dashboard also provides sprint comparison, risk alerts, trend analysis, and graphical insights to support release governance.
The implemented system demonstrates how software delivery metrics can be combined into a practical decision-support framework for Agile release management. The proposed approach improves transparency, supports early risk identification, and enables development teams to make more informed release decisions.
